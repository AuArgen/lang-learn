Тил үйрөнүү үчүн Оюн Платформасы – Техникалык Документация
1. Технологиялык Стек жана Архитектура
Frontend & Backend (BFF): Next.js (App Router). Бул SEO үчүн да, API роуттарды (авторизация, тышкы сурамдар) жазуу үчүн да эң ыңгайлуу.

Стилизация: Tailwind CSS (Mobile-first, адаптивдүү дизайн үчүн).

Маалымат базасы: Firebase Firestore (NoSQL, реалдуу убакыттагы оюндар үчүн).

Аутентификация: Тышкы Google Auth сервиси + Next.js ичиндеги жеке JWT (HTTP-only cookies).

Үн таануу: Web Speech API / Google Cloud Speech-to-Text.

2. Авторизация жана Сессияны Башкаруу (Auth Flow)
Сиздин PHP логикаңыз Next.js'те төмөнкүдөй иштейт:

Кирүү: Колдонуучу "Кирүү" баскычын басканда, тышкы сервис AUTH_SERVICE_URL'ге багытталат (redirect).

Кайтуу (Callback): Тышкы сервис ийгиликтүү болгон соң, платформанын /api/auth/callback?token=... дарегине JWT токен менен кайтарат.

Токенди окуу жана Текшерүү (Next.js API Route):

Келген токен чечмеленет (decode).

Дароо EXTERNAL_CHECK_USER_URL дарегине google_id аркылуу сурам жөнөтүлүп, ролу жана жазылуусунун (subscription) мөөнөтү текшерилет.
Текшеруудо Header ге 'X-API-Key: ' + EXTERNAL_API_KEY кошулат. баары .env файлда сакталат.

Жеке JWT түзүү: Платформа өзүнүн коопсуз JWT токенин (мисалы, jose китепканасы менен) түзөт. Анын ичине колдонуучунун ID'си, ролу (USER, TEACHER, ADMIN) жана жазылуу мөөнөтү камтылат.

Сактоо: Бул жеке JWT браузердин HTTP-only Cookie'сине сакталат. Ошол эле учурда колдонуучунун маалыматтары Firebase users коллекциясына жазылат/жаңыртылат.

3. Колдонуучулардын Ролдору жана Лимиттери
Тутумда 3 негизги роль бар. Ролдор сессия аркылуу көзөмөлдөнөт.

👤 Жөнөкөй колдонуучу (USER - Жазылуусу жок)
Лимит: Болгону 1 тема түзө алат.

Сөз лимити: Ал теманын ичине максимум 3 гана сөз кошо алат.

Бөгөттөө: Лимитке жеткенде жаңы тема же сөз кошуу баскычтары өчүрүлүп (disabled), экранга "Толук мүмкүнчүлүктөрдү колдонуу үчүн жазылуу сатып алыңыз" деген билдирүү жана Жазылуу алуу баскычы чыгат.

Мүмкүнчүлүгү: Мурда жазылуусу бар кезде түзүп койгон темалары жана сөздөрү болсо (мисалы, 100 сөзү бар 5 тема), жазылуусу бүтүп калса да, аларды ойното алат, бирок жаңыны кошо албайт жана өзгөртө албайт.

👨‍🏫 Мугалим (TEACHER - Жазылуусу бар)
Лимит: Чектөөсүз (каалаганча тема жана сөз кошуп озгорто алат).

Публикация: Эгер теманын ичинде 10дон кем эмес сөз болсо, аны жалпыга ачык кылуу үчүн (Публикация) админге сурам жөнөтө алат.

Ачык шилтеме (Public Link): 10дон ашык сөзү бар тема үчүн атайын шилтеме (мисалы: domain.com/play/theme_123) түзө алат. Бул шилтемени окуучуларына жөнөтсө болот.

Аналитика: Өзү түзгөн темалар боюнча ойнолгон бардык оюндардын жыйынтыктарын, окуучулардын упайларын көрө алат.

👑 Администратор (ADMIN)
Лимит: Чектөөсүз (баарын кыла алат).

Модерация: Мугалимдерден келген "Публикацияга сурамдарды" көрөт. Текшерип, уруксат берсе, ал тема "Башкы беттеги жалпы оюндар" бөлүмүнө чыгат.

Өздүк контент: Өзү да тема түзүп, 10дон көп сөз кошуп, эч кандай сурамсыз эле түз Башкы бетке жарыялай алат.

Глобалдык Аналитика: Бардык колдонуучулардын тизмесин, алардын аракеттерин (качан кирди, эмне түздү) жана бардык оюндардын жыйынтыктарын көрө алат.

4. Оюн Логикасы жана Ачык Шилтемелер (Public Games)
Ачык шилтеме (Public Link) аркылуу кирүү:

Окуучу мугалим берген шилтемени басканда, система авторизация талап кылбайт.

Экранда: "Оюнга кошулуу үчүн атыңызды жазыңыз" деген терезе чыгат.

Жеке же командалык режимди тандап, оюнду баштайт.

Базага сактоо: Бул оюндар Firebase'ге сакталып жатканда is_public_link = true деген белги (флаг) менен сакталат. Бул мугалимге "бул оюн шилтеме аркылуу ойнолгон" деп айырмалоого жардам берет.

Башкы беттеги публичный оюндар: Админ тарабынан уруксат берилген же түзүлгөн темалар башкы бетте тизмектелип турат. Каалаган адам кирип ойной алат.

5. Firebase Маалымат Базасынын Структурасы (Firestore Schema)
Бул маалыматтарды туура уюштуруу реалдуу убакытта (real-time) ылдам иштөөнү камсыздайт:

users (Колдонуучулар)

id, google_id, name, email, role, subscription_expires_at, created_at

themes (Темалар)

id, author_id, title, description, words_count, status (draft, pending, published), created_at

words (Сөздөр)

id, theme_id, word, translation, language, audio_url

games (Оюндар/Сессиялар)

id, theme_id, host_id, mode (solo, team), is_public_link (boolean), status (waiting, playing, finished), created_at

game_results (Оюндун жыйынтыктары)

id, game_id, theme_id, player_or_team_name, score, mistakes_count, played_at

6. SEO, Удобство (UI/UX) жана Адаптация
Долбоор элге ачык (публичный) баракчаларды камтыгандыктан, төмөнкүлөр сөзсүз эске алынышы керек:

1. SEO (Издөө системаларын оптималдаштыруу):

Next.js'тин metadata API'син колдонуу. Ар бир жалпыга ачык теманын өзүнүн URL дареги жана сүрөттөлүшү (Title, Description, Open Graph сүрөттөрү) болот. Мисалы кимдир бирөө "Англисче жаныбарлар темасында сөздөрдү жаттоо" деп издесе, сиздин сайт чыгышы керек.

SSR (Server-Side Rendering) колдонулат (публичный баракчалар үчүн), издөө роботтору маалыматты оңой окушу үчүн.

2. Адаптация (Responsive Design):

Tailwind CSS аркылуу мобилдик телефондорго, планшеттерге жана чоң экрандарга (ТВ же проектор) ылайыкташат.

Командалык оюнда (эки экранга бөлүнүү) телефондо үстү-асты, ал эми компьютерде/проектордо сол-оң тарап болуп бөлүнүшү керек.

3. Удобство (UX):

Микроинтеракциялар: Үн жаздыруу башталганда микрофондун пульсацияланган (кыймылдаган) анимациясы.

Таймер: Убакыт азайып баратканда (акыркы 10 секундда) таймер кызарып, визуалдык белги бериши керек.

Ката жөнүндө кабарлоо: Окуучу сөздү туура эмес айтса, дароо "Дагы 2 мүмкүнчүлүк калды" деп жумшак эскертүү чыгышы шарт.
